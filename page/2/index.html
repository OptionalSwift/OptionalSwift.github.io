Why coding like This ------ Optional 揭秘

Optional 揭秘

Topic 1:

请简单写出可选类型的声明方式，以及几种解包形式。

Example:

/// 几种声明方式
var optionalValue : Optional<Int> //完整声明 注意<>中的都是类型
var optionalValue2 : Int?         //显示声明
var optionalValue3 : Int!         //隐式声明

/// 几种解包
/// 演示值
var someOptional:Int? = 3  //初始值为3

/// 1.通过判断是否等于nil 这里并未解包 只能保证安全性
if someOptional != nil{
  println("value is \(someOptional!)")
}

/// 2.if-let绑定
if let value = someOptional{
  println("value is \(value)")
}
/// 3.??解包
println("value is \(someOptional ?? 2)")

/// 4.swift2.0新的匹配模式 但是要在xcode7环境

//新增使用enumeration匹配 也就是枚举匹配方式
if case .Some(let x) = someOptional{
println("someOptional value is \(x)") //如果为nil 情况 可不会输出东西 因为匹配的是Some!
}

//新增使用可选模式匹配

if case let x? = someOptional{
println("someOptional value is \(x)") //不需要对x进行解包之类的东东
}

其中第四种方法仅在xcode7环境 以及swift2.0语法下支持。更多语法改动请点击这里。关于可选类型解包请点击这里。 更多基础语法请查看官方文档。

tableview and collectionView 学习总结

tableView

  1. tableview.registerClass(UITableViewCell.classForCoder(),forCellReuseIdentifier:"cell") 其中第一个参数UITableViewCell.classForCoder()返回的是类名,其次这个函数是在注册cell

  2. tableView主要有dataSource 和 delegate 两个协议。 前者是提供数据，后者是一些处理方法。显然我不会讲numberOfSectionInTableView、tableView:numberOfRowInSection和tableView:cellForRowAtIndexPath。而是了解到了构建自定义Header 和 Footers(ps：当然也可以通过dataSource 中的tableView:titleForHeaderInSection以及tableView:titleForFooterInSection)，采用tableView:ViewForHeaderInSection和tableVIew:ViewForFooterInSection，这两个方法可不是dataSource中的，而是delegate中的方法，这是让我很不解的地方，也是曾经我和Jim争论过的问题。
