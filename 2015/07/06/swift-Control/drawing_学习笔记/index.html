UIImage

  1. Asset catalog 和 Top level of app bundle 两个地方存储image。
  2. 调用init(named:)方法，首先会从Asset catalog找起，其次是Top level of app bundle。
  3. init(named:)方法有利有弊：由于会缓存到memory中，意味着一旦图片多，吃的内存就多；但是也正因为缓存到内存中，之后的调用会非常迅速！
  4. 使用init(contentsOfFile:)方法得到图片的同时不会缓存，但是你必须定位到图片的path,NSBundle.mainBundle()方法得到app’s bundle，然后使用实例方法pathForResource:ofType来得到图片
  5. 对于double-resolution 或者 triple-resolution的device,假如有@2x后缀的图片会自动调用，否则嘛会设置scale property = 2.0。iPhone中 one Point 可能包含1、2、3pixels。
  6. Asset catlog 也有sizeClass!

  7. 通过imageWithTraitCollection获得图片的trait collection，traitCollectionDidChanged则是实时反馈属性变换(例如屏幕旋转)

  8. 代码生成UIImageAsset，设置什么图片在什么tc下显示，加载到asset catalog，真实是没有asset catalog的

UIImageView

  1. media Libary 可以通过快捷键(command-option-control-4)
  2. UIImageView 有两个图片：image和highlightedImage，由highlighted 属性来标识。
  3. UIImageView 的 默认contentMode为.ScaleToFill，文中提到一开始是view来适应图片的大小(有待考证)
  4. 在SB中放置一个imageView，赋值给它一张图片让其显示，这时候ImageView的大小尺寸去决定于他是否使用了autolayout。没有则imageView 本身是多少就是多少，假如使用了autolayout 那么Imageview就会反过来适应图片的size,除非你给imageView加了约束

Resible Images

  1. resizingMode: 有.Tile、.Stretch 两种
  2. 产生一个resibleimage 通过resizableImageWithCapInsets()方法

Image Rendering Mode

1.UIImage RenderingMode 分为三个：.Automatic .AlwaysOriginal .AlwaysTemplate

Graphics Context

  1. 只有手头有context 才可以draw.
  2. context怎么得到。UIGraphicsBeginImageContextWithOptions()创建context并设置为当前的context;绘画完全使用UIGraphicGetImageFromCurrentImageContext()得到先前绘制好的图形；最后使用UIGraphicsEndImageContext去除context。
  3. 继承UIView ，然后在drawRect方法中绘制，这里有意思的是Cocoa 已经创建好一个Graphic context作为当前的context供你使用了，
  4. 但是假如subclass CALayer 实现drawInContext：incontext方法 你必须先把这个incontext作为当前画布，才能进行绘画。UIGraphicPushContext(与之相对的是UIGraphicPopContext).
  5. UIKit中 有很多Cocoa Class 知道如何去绘制自己，比如：UIImage、NSString(文字喽)、UIBezierPath（形状），UIColor.在UIKit中 我们只能绘制到当前的context中，否则没有效果的，那么我们必须使用UIGraphicsBeginImageContextWithOptions()或者drawRect
  6. Core Graphics 这个貌似是完整的绘画API,UIKit中的draw也是封装这个的。
  7. 自然我们也要制定一个context,叫做CGContext。CoreGraphic 我们必须首先生成context 设为当前的context,Core Graphic 还需要去reference 到它才能用，使用方法UIGraphicsGetCurrentContext

  • 1.UIKit drawRect方法

  • 2.Core Graphics drawRect方法,context已经存在，我们要做的是指向它

  • 3.UIKit drawLayer方法

  • 4.Core Graphics drawLayer方法

  • 5.UIKit的 UIImageView

  • 6.Core Graphic UIImageView

总结：由于UIKit是对Core Graphic的封装，预先为我们准备好了context,并指向它了，所以我们只需要用UIBezierPath等都是在context上操作的；而Core graphic，必须要指向当然的current context，由于UIkit已经生成了，那么我们只需要UIgraphicgetCurrentContext即可，仔细观察会发现Core Graphic操作方法都是要把context作为参数传入操作的。

UIImage Drawing

  1. UIImage的绘制，同样先创建一个context。使用方法UIGraphicBeginImagContextwithOptions()，之后就是绘制，然后从context提取出绘制好的图片image,反馈给imageView的image.
  2. 注意两个方法drawAtPoint和drawInRect

CGImage Drawing

  1. UIImage的Core Graphic版本就是CGImage.
  2. 手头有CGImage 用init(CGImage:)转换到UIimage,另外还有init(CGImage:scale:orientation)
  3. 有个猜测 是不是所有的UIimage 都是由CGImage得来的呢，然后UIImage中的CGImgae属性就是原始那个CGImage呢？
  4. 238页例子会有一些问题，首先是图形上下颠倒了，主要原因还是在于创建完CGImage，用CGContextDrawimage时候发生的上下颠倒，文章指出是市区了native 坐标系产生的，简单的处理是重新绘制一次即可，我理解为负负得正。
  5. 第二个问题是在high-resolution device中，绘制会出现一些问题。
  6. image.size单位是Point,比如图片名字叫Mars.png、Mars@2x.png,Mar@3x.png 在double-resolution 设别上scale =2 ,在tripl设备上scale=3。通过UIimage(named:)方法得到的image 会按照设备来获得的，意味着double device上会获得2x图片 triple device获得3x图片。 但是size 都是Mars.png的size。
